깃 따라하기~~
vs코드에서는 터미널 열때 ctrl+`하면 된다.

깃으로 추가, 변경, 삭제된 모든 파일을 스테이징할때
git add . 
or
git add *

나는 git add . 만 썼었는데 굿~

git status 하면 변경사항이 일어난 모든 파일이 보이지롱~
스테이징된 파일은 초록색으로, 스테이징 되지 않은 파일은 빨간색 글자로 보인다

스테이징 됐다고 하더라도 커밋하지 않거나, 스테이징 된 이후 변경된 파일의 내용은 적용이 안된다~

커밋할때는 git commit -m "커밋내용"을 적으면 된다

커밋을 할때 커밋내용을 잘 작성해줘야
커밋 자체는 로컬에 적용하는거고
커밋내용 변경사항도 알 수 있다.

물론 커밋내용을 직접 보고 확인해도 되지만, 잘 적어줘야 편하다

커밋내용을 작성하는 규칙도 있다.

뭘 했는지 커밋제목으로 작성해주고
어떤 동작을 하는지 간단 명료하게 작성을 해줘야한다

git diff 파일경로/파일명.파일확장자 로 알 수 있다.

git push는 github 에 업로드하는 명령어인데
git push 명령어는 git remote에서 확인되는 주소만 가능한데
처음 로컬에서 git init을 하고난 뒤에는 추가된 저장소가 없기 때문에
git remote add 저장소주소 저장소이름 
명령어로 추가해줘야한다.

그 뒤에 git push 원격저장소이름 로컬브랜치이름 으로 깃허브에 푸시가 가능하고
당겨오고 할 수 있다는것.

이번에 명확하게 머리에 박힌건
저장소마다 git config 로 사용자의 이름과 이메일주소 등을 등록할수있다는것.
